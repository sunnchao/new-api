本次设计要确认的边界
后端： 当前仓库里的 token 接口基本都是“当前用户自己的 token”。
所以如果管理员要“看全站 + 管所有用户”，需要补一套独立的 /api/admin/token/* 路由、controller 和 model 查询，不只是改前端地址。

前端： 管理员页面可以继续复用“我的令牌”的表格、筛选、样式， 但操作按钮不一定应该 100% 复用。比如“显示密钥”“聊天”“批量删除”这种动作， 是否对管理员开放，需要单独定规则。

多语言： 新增的“我的令牌”“管理员令牌管理”“用户 ID”“用户名”等文案， 需要同步补到所有 locale 文件里，不能只改 React 里的 t('中文')。
